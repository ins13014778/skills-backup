# 漏洞统一分级标准（PHP）

---

## 1. 严重等级计算公式

```
Score = R × 0.40 + I × 0.35 + C × 0.25
CVSS 映射 = Score / 3.0 × 10.0
说明：
- R/I/C 为三维离散值（0-3）
- CVSS 建议保留 1 位小数
```

---

## 2. 三维评分标准

### 可达性 (Reachability, R)
- 3：无需认证，HTTP 直接可达（或 `AUTH` 分级为“❌ 无鉴权”）
- 2：需要普通用户认证（或 `AUTH` 分级为“✅ 仅认证/⚠️ 仅认证 P1”）
- 1：需要管理员权限或内网访问（或需要特定资源所有权/角色才可触达）
- 0：不可达（死代码/trace_status=UNRESOLVED/被强有效防护拦截导致无法进入敏感语义）

### 影响范围 (Impact, I)
- 3：RCE/任意文件写入/完全数据泄露/系统沦陷
- 2：敏感数据泄露/越权操作/部分文件读取/高价值业务影响
- 1：有限信息泄露/低影响配置读取/局部可见性缺陷
- 0：无实际安全影响（仅理论风险或已被隔离）

### 利用复杂度 (Complexity, C)
- 3：低复杂度（单次请求即可用；payload 直达 sink）
- 2：中复杂度（需构造 payload、多步操作或存在利用前置条件但常规环境可完成）
- 1：高复杂度（需要特定环境/竞态/链式利用/需要特定数据状态）
- 0：不可利用（存在有效防护且绕不过；或关键 sink 不能被实际执行）

## 4. 漏洞编号规范

格式：`{等级前缀}-{漏洞类型}-{序号}`

示例：
- `C-SQL-001` SQL 注入（Critical）
- `H-AUTH-002` 鉴权绕过/越权（High）
- `M-FILE-003` 文件读取（Medium）

漏洞类型代码建议（PHP）：
- `SQL` / `NOSQL` / `CMD` / `SSRF` / `XSS` / `FILE` / `UPLOAD` / `WRITE` / `ARCHIVE` / `REDIR` / `CRLF`
  / `XXE` / `DESER` / `TPL` / `LDAP` / `EXPR`
  / `AUTH` / `CSRF` / `SESS` / `CFG`
  / `CRYPTO` / `LOGIC` / `FS` / `LOG`

---

## 5. 可利用性标注

| 可利用性 | 含义 | 对分级的影响 |
|----------|------|----------------|
| ✅ 已确认可利用 | 代码路径在目标条件下可执行，且可利用性成立 | 按你估计的 `R/I/C` 计算 `Score` 与 CVSS |
| ⚠️ 待验证 | 存在风险但关键条件未确认（常见：trace_status=PARTIAL 或关键证据点缺失） | 默认 `C = max(C-1, 0)`，并在报告中说明缺失证据对应的前置条件 |
| ❌ 不可利用 | 有效防护，绕不过（或关键 sink 未实际执行） | 强制 `R=0 且 C=0`，因此 `Score=0`，CVSS 映射为 0 |
| 🔍 环境依赖 | 需要特定环境/组件/网络条件才能利用 | 默认 `R=max(R-1,0)` 与 `C=max(C-1,0)`，并说明依赖条件 |
